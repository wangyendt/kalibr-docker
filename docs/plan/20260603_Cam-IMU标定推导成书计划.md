# Cam-IMU 标定推导成书计划

## 背景

现有 knowhow 已经把 Kalibr cam-imu 标定中的 residual 和 Jacobian 按源码结果展开出来，但阅读体验更像“对照答案解释答案”。如果读者还没有先建立问题图景、符号系统、坐标约定和优化直觉，很容易在公式里迷路。

下一版文档要改成一本小书的写法：先解释为什么要这样构建问题，再一点点引出状态量、观测、预测模型、残差、权重、Jacobian 和源码实现。目标不是一次性堆完全部公式，而是每章只解决一个核心问题，并让前一章自然成为后一章的前提。

## 核心问题

这本书要回答一个问题：

> 给定相机看到的标定板角点和 IMU 的加速度/角速度测量，Kalibr 为什么能同时估计相机-IMU 外参、时间偏移、连续运动轨迹、IMU bias 和重力？这些 residual 和 Jacobian 是如何一步步从几何和刚体运动推出来的？

这个问题不能从源码函数名开始讲。它要从“我们到底在比较什么”开始讲：

- 相机比较的是：标定板上一个已知三维点投到图像后，和检测到的像素点差多少。
- 陀螺仪比较的是：连续轨迹导出的 body angular velocity 经过 IMU 外参和 bias 后，和 IMU 读数差多少。
- 加速度计比较的是：连续轨迹二阶导、重力、旋转、lever arm 刚体加速度补偿合成的 specific force，和加速度计读数差多少。
- 优化器比较的是：所有 residual 被 covariance 和 robust kernel 加权后，整体平方和是否最小。

## 目标

1. 写成“从零开始”的工程数学叙事，而不是已有公式的解释。
2. 每章只引入必要符号，符号全局一致，不在不同章节临时换记法。
3. 每个 residual 都按同一节奏展开：
   - 直觉：它在比较什么。
   - 物理/几何模型：预测量怎么来。
   - 残差方向：为什么是 `measurement - prediction` 或 `prediction - measurement`。
   - 加权方式：covariance、information matrix、robust kernel 如何进入。
   - Jacobian：先对中间量求，再用链式法则推到设计变量。
   - 源码对应：代码里哪几行构建了同一个东西。
4. 书稿和源码双向可查：读公式能找到代码，读代码能回到公式。
5. 推导中明确区分三层：
   - 官方 Kalibr 数学模型。
   - 本仓库当前实现。
   - 本地工程改动对目标函数的影响。

## 符号系统策略

符号表可以最后统一补全，但符号规则必须从第一章开始固定。后续书稿采用两层语言：

- 理论主语言：以 `/Users/wayne/Downloads/A_micro_Lie_theory_for_state_estimation_in_robotics.pdf` 的 Micro Lie Theory 记号为主，使用 Lie group、tangent vector、$\mathrm{Exp}$、$\mathrm{Log}$、$\oplus$、$\ominus$、$\mathrm{Ad}$，并显式区分 right/local 与 left/global tangent。
- 代码桥接语言：当公式落到 Kalibr 源码时，默认使用 Kalibr 的左侧/global 扰动约定 $\boxplus_K$。它与 Micro 标准 left-plus 的关系是旋转增量取负号：$\delta\boldsymbol\phi_L=-\delta\boldsymbol\phi_K$。

写作顺序也随之调整：先用 Micro Lie Theory 讲清楚 right/local 与 left/global 的区别，再把 Kalibr 的 $\boxplus_K$ 作为后续 Jacobian 主推导约定。这样读源码时不需要每一步都从 right-plus 额外换号。

### 坐标系

| 符号 | 含义 |
|---|---|
| `w` | world / calibration target 初始化后的世界系 |
| `b` | body frame，Kalibr 中通常取参考 IMU body |
| `i` | 某个 IMU frame |
| `c_n` | 第 `n` 个 camera frame |
| `m` | marker / calibration target frame |
| `t` | 时间标量，只用于时间，不作为 frame 下标 |

### 变换

刚体位姿统一视为 Lie group 元素，并采用 Micro Lie Theory 风格的粗体记号：

$$
\mathbf T_{ab}\in SE(3).
$$

含义仍然是：

$$
\mathbf T_{ab}: \text{把 } b \text{ 系坐标表示的点变到 } a \text{ 系。}
$$

也就是：

$$
\mathbf p_a = \mathbf T_{ab}\cdot \mathbf p_b,\qquad
\mathbf R_{ab}\in SO(3),\qquad
\mathbf p_a=\mathbf R_{ab}\mathbf p_b+\mathbf t_{ab}.
$$

在源码对应段落里允许出现 Kalibr 代码变量 `T_ab`，但第一次出现时必须说明：代码里的 `T_ab` 对应书稿中的 $\mathbf T_{ab}$，二者只是记号差异。

Kalibr 代码中的关键对象：

| 代码 | 书稿符号 | 含义 |
|---|---|---|
| `T_w_b` | $\mathbf T_{wb}$ | body 到 world |
| `T_b_w` | $\mathbf T_{bw}=\mathbf T_{wb}^{-1}$ | world 到 body |
| `T_cN_b` | $\mathbf T_{c_nb}$ | body 到 camera |
| `T_c_w` | $\mathbf T_{c_nw}=\mathbf T_{c_nb}\mathbf T_{bw}$ | world 到 camera |
| `C_b_w` | $\mathbf R_{bw}$ | world 向量转 body |
| `C_i_b` | $\mathbf R_{ib}$ | body 向量转 IMU |
| `r_b` | $\mathbf r_b$ | IMU lever arm，表达在 body frame |

### Lie group 扰动

先列 Micro Lie Theory 的标准 right/local 与 left/global：

$$
\mathbf T\oplus_R{}^{\mathbf T}\boldsymbol\tau
=
\mathbf T\circ\mathrm{Exp}({}^{\mathbf T}\boldsymbol\tau),
\qquad
\mathbf S\ominus_R\mathbf T
=
\mathrm{Log}(\mathbf T^{-1}\circ \mathbf S).
$$

$$
{}^E\boldsymbol\tau\oplus_L\mathbf T
=
\mathrm{Exp}({}^E\boldsymbol\tau)\circ\mathbf T,
\qquad
\mathbf S\ominus_L\mathbf T
=
\mathrm{Log}(\mathbf S\circ\mathbf T^{-1}).
$$

其中：

- $\mathbf T,\mathbf S\in\mathcal M$ 是 Lie group 元素。
- ${}^{\mathbf T}\boldsymbol\tau$ 是 local tangent。
- ${}^E\boldsymbol\tau$ 是 global tangent。
- $\mathrm{Exp}:\mathbb R^m\rightarrow\mathcal M$。
- $\mathrm{Log}:\mathcal M\rightarrow\mathbb R^m$。

local tangent 和 global tangent 通过 adjoint 关联：

$$
{}^E\boldsymbol\tau = \mathrm{Ad}_{\mathbf T}\,{}^{\mathbf T}\boldsymbol\tau.
$$

后续若写优化更新，优先写 Kalibr convention：

$$
\mathbf T^+ = \mathbf T\boxplus_K\boldsymbol\delta.
$$

其中旋转部分满足：

$$
\mathbf R\boxplus_K\delta\boldsymbol\phi
=
\mathrm{Exp}(-\delta\boldsymbol\phi)\mathbf R.
$$

点 action 的局部线性化满足：

$$
\delta(\mathbf T\cdot\mathbf p)
=
\delta\boldsymbol\rho
+
[\mathbf T\cdot\mathbf p]_\times\delta\boldsymbol\phi.
$$

若必须讨论 Micro 标准 left-plus 或 right-plus，必须明确写出转换关系，不能省略。

### 残差方向

全书固定写法：

| residual | 数学方向 | 代码依据 |
|---|---|---|
| camera | $\mathbf e_\pi=\mathbf y-\hat{\mathbf y}$ | `ReprojectionError` |
| gyro | $\mathbf e_\omega=\hat{\boldsymbol\omega}-\mathbf z_\omega$ | `EuclideanError` |
| accel | $\mathbf e_a=\hat{\mathbf a}-\mathbf z_a$ | `EuclideanError` |
| Euclidean prior | $e=x-x_0$ | `ErrorTermEuclidean` |
| Transformation prior | $\mathbf e_T=\mathrm{Log}(\mathbf T\mathbf T_0^{-1})$ 的 Kalibr 实现 | `ErrorTermTransformation` |

### Jacobian 方向

文档里要显式区分：

- 普通微分：$\delta y=J_x\delta x$。
- Micro Lie Theory right/local 和 left/global Jacobian：用于解释 tangent space 的表达位置。
- Kalibr Jacobian：默认通过 $\mathbf T\boxplus_K\boldsymbol\delta$ 定义扰动。
- Lie group 局部扰动：$\delta\mathbf T$ 或 $\delta\boldsymbol\phi$ 不是矩阵元素微分，而是 tangent vector。
- Kalibr 的 `sm_kinematics` 小角度号约定，与 Micro Lie Theory 的标准 SO(3) left-plus 满足 $\delta\boldsymbol\phi_L=-\delta\boldsymbol\phi_K$。

所有 Lie 相关推导先写 Micro 形式，再给 Kalibr 桥接表。桥接表必须覆盖：

- composition：$\mathbf T\circ \mathbf S$。
- inverse：$\mathbf T^{-1}$。
- action：$\mathbf T\cdot \mathbf p$ 或 $\mathbf R\mathbf p$。
- plus/minus：$\oplus_R,\ominus_R,\oplus_L,\ominus_L,\boxplus_K$。
- adjoint：$\mathrm{Ad}_{\mathbf T}$。
- rotation action 的符号差异。

### 第 1 章：问题从哪里来

核心问题：为什么 cam-imu 标定可以写成优化问题？

叙事线：

1. 相机只能看到图像点。
2. IMU 只能测到自身角速度和 specific force。
3. 标定板提供已知三维几何。
4. 连续轨迹把不同时间的相机和 IMU 观测连接起来。
5. 外参和时间偏移决定“相机看到的运动”和“IMU 感到的运动”是否一致。

要讲清楚：

- 状态量为什么包含轨迹、外参、时间偏移、bias、重力。
- 观测为什么分成 camera / gyro / accel 三类。
- residual 是“预测观测”和“真实观测”的差。
- 所有 residual 放在一起就是 weighted nonlinear least squares。

本章不做复杂 Jacobian，只建立直觉。

### 第 2 章：坐标系与刚体变换

核心问题：一个点如何从标定板走到相机像素？

叙事线：

1. 点只有在坐标系里才有数值。
2. $\mathbf T_{ab}$ 表示“从 b 到 a”的 $SE(3)$ group element。
3. 旋转和平移组合成 SE(3)。
4. 逆变换为什么会改变方向。
5. camera residual 中的链路：

$$
\mathbf p_{c_n}
=\mathbf T_{c_nb}\mathbf T_{bw}\cdot \mathbf p_w.
$$

要推导：

- $\mathbf T_{ab}^{-1}=\mathbf T_{ba}$。
- $\mathbf T_{ac}=\mathbf T_{ab}\mathbf T_{bc}$。
- 点变换的一阶扰动。
- `T_w_b.inverse()` 和 `T_cN_b * T_b_w` 的代码含义。

### 第 3 章：最小二乘与 residual 方向

核心问题：一个 residual 的 error、weight、Jacobian 如何进入 Gauss-Newton 线性系统？

叙事线：

1. 单个 residual vector 只能说明一个观测不匹配，还不能直接和其他 residual 相加。
2. covariance 告诉优化器这个 residual 可信度有多高。
3. information matrix 是 covariance 的逆，square-root information 把加权最小二乘写成普通最小二乘。
4. residual 方向决定 Jacobian 的符号：camera 是 measurement minus prediction，IMU Euclidean residual 是 prediction minus measurement。
5. Gauss-Newton 把 whitened residual 和 whitened Jacobian 组装成 $H\delta\theta=b$。
6. robust kernel 不是改 residual 定义，而是在当前线性系统中改大 residual 的权重。

要推导：

$$
E(\theta)=\sum_k \mathbf e_k(\theta)^\top \boldsymbol\Omega_k\mathbf e_k(\theta),
\qquad
\boldsymbol\Omega_k=\boldsymbol\Sigma_k^{-1}.
$$

square-root information：

$$
\boldsymbol\Omega_k=\mathbf L_k\mathbf L_k^\top,
\qquad
\bar{\mathbf e}_k=\mathbf L_k^\top\mathbf e_k,
\qquad
\bar{\mathbf J}_k=\mathbf L_k^\top\mathbf J_k.
$$

以及线性化：

$$
\mathbf e(\theta\boxplus_K\boldsymbol\delta)
\approx \mathbf e(\theta)+\mathbf J\boldsymbol\delta.
$$

正规方程：

$$
\mathbf H
=
\sum_k\mathbf J_k^\top\boldsymbol\Omega_k\mathbf J_k,
\qquad
\mathbf b
=
-
\sum_k\mathbf J_k^\top\boldsymbol\Omega_k\mathbf e_k.
$$

使用 robust kernel 时，当前迭代中：

$$
\mathbf H\mathrel{+}=w_k\mathbf J_k^\top\boldsymbol\Omega_k\mathbf J_k,
\qquad
\mathbf b\mathrel{+}=-w_k\mathbf J_k^\top\boldsymbol\Omega_k\mathbf e_k.
$$

本章还要和第 0 章的本地差异表保持一致：

- `R * 2` 是代码层变量名，数学上改的是 information weight / covariance 假设。
- Huber / Cauchy / Blake-Zisserman 改的是 robust weighting。
- 这些不是 raw residual 或 raw Jacobian 的符号变化。
- 本章不负责推完所有具体 Jacobian；第 4-10 章逐个 residual 推。

### 第 4 章：相机观测模型

核心问题：为什么 camera error 是 $\mathbf y-\pi(\mathbf T_{cw}\cdot \mathbf p_w)$？

叙事线：

1. 标定板上点 $\mathbf p_w$ 已知。
2. 当前时刻 body pose 给出 $\mathbf T_{wb}(t)$。
3. 外参给出 $\mathbf T_{cb}$。
4. 组合得到点在相机下的坐标。
5. 相机模型投影到像素。
6. 像素测量和预测像素相减。

要推导：

$$
\hat{\mathbf y}
=\pi\left(\mathbf T_{c_nb}\mathbf T_{bw}(t)\cdot \mathbf p_w\right),
\qquad
\mathbf e_\pi=\mathbf y-\hat{\mathbf y}.
$$

Jacobian 分层：

$$
\frac{\partial e_\pi}{\partial \theta}
=
-J_\pi
J_{p_c}
J_{\mathbf T_{c_nb}\mathbf T_{bw}}
J_\theta.
$$

重点解释负号来自 $y-\hat y$，不是 Lie convention。

### 第 5 章：连续时间轨迹与 B 样条

核心问题：为什么不用每帧一个 pose，而要用 continuous-time spline？

叙事线：

1. 相机低频、IMU 高频，时间不对齐。
2. IMU residual 需要速度和加速度。
3. 离散 pose 差分会带来噪声和插值问题。
4. B-spline 给出任意时刻的 pose、速度和加速度。

要推导：

- B-spline basis 如何把局部控制点线性组合成曲线值。
- 位置的一阶、二阶导。
- 旋转向量曲线如何导出 body angular velocity。
- 时间偏移的 Jacobian 为什么是对时间求导。

核心公式：

$$
c(t)=\sum_j B_j(t)c_j,
\qquad
\dot c(t)=\sum_j \dot B_j(t)c_j,
\qquad
\ddot c(t)=\sum_j \ddot B_j(t)c_j.
$$

### 第 6 章：陀螺仪 residual

核心问题：陀螺仪测量和视觉轨迹导出的角速度为什么能比较？

叙事线：

1. 刚体上所有点角速度相同。
2. pose spline 的旋转部分可以导出 body frame angular velocity。
3. 如果 IMU frame 和 body frame 不重合，要旋转到 IMU frame。
4. 加上 gyro bias 得到预测测量。

普通 IMU 模型：

$$
\hat{\boldsymbol\omega}_i
=\mathbf R_{ib}\boldsymbol\omega_b+\mathbf b_g(t),
\qquad
\mathbf e_\omega=\hat{\boldsymbol\omega}_i-\mathbf z_\omega.
$$

要推导：

- 对 $\mathbf R_{ib}$ 的 Jacobian。
- 对 $\boldsymbol\omega_b$ 的 Jacobian。
- 对 bias spline 控制点的 Jacobian。
- 对 pose spline 控制点的链式 Jacobian。
- 为什么 `EuclideanError` 没有 camera residual 的负号。

### 第 7 章：加速度计 residual

核心问题：加速度计测的不是世界加速度本身，而是什么？

叙事线：

1. 加速度计测 specific force。
2. 世界加速度需要减重力。
3. 转到 body frame 后，如果 IMU 不在 body 原点，还要加刚体 lever arm 项。
4. 再转到 IMU frame，加 bias。

普通 IMU 模型：

$$
\hat{\mathbf a}_i
=
\mathbf R_{ib}
\left(
\mathbf R_{bw}(\mathbf a_w-\mathbf g_w)
+\dot{\boldsymbol\omega}_b\times \mathbf r_b
+\boldsymbol\omega_b\times(\boldsymbol\omega_b\times \mathbf r_b)
\right)
+\mathbf b_a(t).
$$

要推导：

- $\mathbf R_{bw}(\mathbf a_w-\mathbf g_w)$ 的 Jacobian。
- $\dot{\boldsymbol\omega}\times \mathbf r$ 的 Jacobian。
- $\boldsymbol\omega\times(\boldsymbol\omega\times \mathbf r)$ 的 Jacobian。
- 对 $\mathbf r_b$、$\mathbf R_{ib}$、$\mathbf g_w$、bias、spline 控制点的 Jacobian。

本章必须非常慢，因为这是最容易看不懂的一章。

### 第 8 章：时间偏移与外参

核心问题：为什么时间偏移也能被优化？

叙事线：

1. 图像时间戳和 IMU 时间戳可能有 offset。
2. camera residual 里的 pose 是 $\mathbf T_{wb}(t_\text{cam}+\Delta t)$。
3. 当 $\Delta t$ 改变，取样的 spline pose 也改变。
4. 所以时间偏移的 Jacobian 来自 pose 对时间的导数。

核心公式：

$$
\frac{\partial e_\pi}{\partial \Delta t}
=
\frac{\partial e_\pi}{\partial \mathbf T_{wb}}
\frac{\partial \mathbf T_{wb}(t)}{\partial t}.
$$

同时讲清楚外参：

- $\mathbf T_{cb}$ 是 camera residual 的直接设计变量。
- $\mathbf R_{ib}, \mathbf r_b$ 是 IMU residual 的直接设计变量。
- 多相机 chain 通过相邻 camera baseline 传递外参。

### 第 9 章：bias spline 与 motion prior

核心问题：为什么 bias 不是常量，也不能完全自由？

叙事线：

1. IMU bias 会慢慢漂移。
2. 用 spline 表示 bias 可以描述慢变化。
3. 但没有约束的 bias spline 会吞掉运动误差。
4. random walk prior 给 bias 的“运动”加惩罚。

要讲清楚：

- bias residual 和 IMU residual 的关系。
- `BSplineMotionError` 是二次型，不是普通 residual vector。
- 为什么它直接贡献 Hessian / RHS。

### 第 10 章：扩展 IMU 模型

核心问题：scale-misalignment 和 size-effect 到底在普通模型上加了什么？

叙事线：

1. 普通模型假设 IMU 三轴正交且尺度准确。
2. scale-misalignment 用矩阵 $M$ 修正轴间耦合和尺度。
3. gyro 的 acceleration sensitivity 用矩阵 $A$ 把加速度耦合到角速度读数。
4. size-effect 允许三个加速度轴有不同 lever arm。

要推导：

$$
\hat{\mathbf a}_i=\mathbf M_a \mathbf R_{ib}\mathbf a_b+\mathbf b_a.
$$

$$
\hat{\boldsymbol\omega}_g
=\mathbf M_g \mathbf R_{gb}\boldsymbol\omega_b
+\mathbf A_g \mathbf R_{gb}\mathbf a_b
+\mathbf b_g.
$$

以及 size-effect 的三个轴向 lever arm。

### 第 11 章：表达式图与源码对应

核心问题：Kalibr 代码为什么看不到手写完整 Jacobian？

叙事线：

1. Python 只搭 expression graph。
2. C++ expression node 提供局部 Jacobian。
3. JacobianContainer 自动按链式法则传到设计变量。
4. 所谓“所有 Jacobian”其实分布在 expression primitives 里。

要整理的 primitive：

- `EuclideanExpressionNodeAdd/Subtract`
- `EuclideanExpressionNodeCrossEuclidean`
- `EuclideanExpressionNodeMultiplyRotation`
- `EuclideanExpressionNodeMultiplyMatrix`
- `HomogeneousExpressionNodeTransform`
- `TransformationExpressionNodeMultiply`
- `TransformationExpressionNodeInverse`
- `BSplineExpressionNode`
- `TransformationTimeOffsetExpressionNode`

每个 primitive 都要给出：

- 数学形式。
- 局部 Jacobian。
- 符号方向。
- 源码位置。

### 附录 B：数值验证与最小复现

核心问题：怎么知道符号没错？

内容：

- 用 Python/C++ 最小实现复现核心公式，而不是复现完整 Kalibr。
- 先实现 SO(3)/SE(3)、projection、residual 和 analytic Jacobian。
- 再用 finite difference 验证每个 analytic Jacobian。
- 对单个 expression node 做有限差分。
- 对 camera residual 做外参扰动检查。
- 对 gyro residual 做 bias / rotation 扰动检查。
- 对 accel residual 做 gravity / lever arm 扰动检查。
- 常见负号错误：
  - $\mathbf T_{ab}$ 方向搞反。
  - camera residual 的负号忘记。
  - Kalibr rotation perturbation 和 Micro Lie Theory 标准 left/right perturbation 混用。
  - specific force 写成 $a+g$ 或 $a-g$ 时坐标系没说明。
