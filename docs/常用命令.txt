# Kalibr Docker 常用命令

以下命令都保持为单行，方便直接复制执行。命令里的 `/ABS/...` 是调用机器上的绝对路径，需要替换成真实路径；Docker 容器内统一使用 `/input`、`/target.yaml`、`/output`、`/data` 这类短路径。

======================================================================
Docker 镜像构建、检查、清理
======================================================================

说明：在 kalibr 仓库根目录构建本项目的 Ubuntu 20.04 / ROS Noetic / Kalibr Docker 镜像；首次构建需要联网下载依赖，后续只要 Docker 镜像还在，本机可以直接运行标定命令。
cd /Users/wayne/Documents/work/code/project/ffalcon/kalibr && docker build -f docker/camera-calibration/Dockerfile -t kalibr-camera-calibration:20.04 .

说明：显式指定 vio_common 的分支、tag 或 commit 构建镜像；当需要固定 bag 生成逻辑时使用，避免 GitHub 默认分支变化导致不同机器构建出来的镜像行为不一致。
cd /Users/wayne/Documents/work/code/project/ffalcon/kalibr && docker build --build-arg VIO_COMMON_REF=master -f docker/camera-calibration/Dockerfile -t kalibr-camera-calibration:20.04 .

说明：查看本机是否已经存在可运行的标定镜像；如果没有 `kalibr-camera-calibration 20.04`，需要先 build、docker load 或从镜像仓库 pull。
docker images kalibr-camera-calibration:20.04

说明：查看当前 Docker 磁盘占用，包括镜像、容器、volume 和 build cache；清理前后都建议执行一次，确认是不是 build cache 占了空间。
docker system df

说明：删除所有未被使用的构建缓存；适合 Dockerfile 多次构建后释放空间，不会删除当前仍保留的 `kalibr-camera-calibration:20.04` 镜像。
docker builder prune -af

说明：删除所有 dangling images，也就是 `<none>:<none>` 的中间镜像；适合构建失败或反复改 Dockerfile 后清理无用 image layer。
docker image prune -f

说明：把当前镜像导出成 tar 文件，适合没有 DockerHub/GHCR 权限的 Windows 或 Ubuntu 机器离线使用；目标机器执行 docker load 后就能运行同一个镜像。
docker save kalibr-camera-calibration:20.04 -o kalibr-camera-calibration_20.04.tar

说明：在另一台机器导入离线镜像 tar；导入后用 `docker images kalibr-camera-calibration:20.04` 确认镜像存在。
docker load -i kalibr-camera-calibration_20.04.tar

说明：从 DockerHub 拉取公开镜像的推荐形式；如果镜像发布到 DockerHub，外部用户只需要 pull 一次，之后标定命令就是一行 docker run。
docker pull wangyendt/kalibr-camera-calibration:20.04

说明：从 GitHub Container Registry 拉取公开镜像的推荐形式；如果项目代码托管在 GitHub，GHCR 通常比 DockerHub 更方便和仓库版本对应。
docker pull ghcr.io/wangyendt/kalibr-camera-calibration:20.04

======================================================================
帮助与参数查看
======================================================================

说明：查看总入口帮助，确认镜像能启动，并看到 `cam-cam` 和 `cam-imu` 两个子命令；这是最轻量的健康检查命令。
docker run --rm kalibr-camera-calibration:20.04 --help

说明：查看 cam-cam 标定参数，包括输入目录、target yaml、输出目录、相机模型、resize、预处理、verbose、skip-kalibr 等。
docker run --rm kalibr-camera-calibration:20.04 cam-cam --help

说明：查看 cam-imu 标定参数，包括 target、cam-chain、imu-yaml、corner-file/H5 两种输入模式、fixture-id、时间偏移估计和报告输出。
docker run --rm kalibr-camera-calibration:20.04 cam-imu --help

======================================================================
cam-cam：单相机、视频、双相机、三相机
======================================================================

说明：单相机图片文件夹标定；`/ABS/images` 目录下直接放 png/jpg/jpeg/bmp/tif 图片，wrapper 会自动整理成 Kalibr `mav0/cam0` 格式、生成 bag、运行单相机内参标定并在 `/ABS/output` 写报告。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --lang zh

说明：单相机视频标定；`/ABS/video.mp4` 会按 `--video-sample-fps` 抽帧，适合只有一个视频的场景，默认仍按单相机处理，输出目录包含抽帧后的 dataset、bag、Kalibr 日志和报告。
docker run --rm -v /ABS/video.mp4:/input/video.mp4:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output:/output kalibr-camera-calibration:20.04 cam-cam --input /input/video.mp4 --target /target.yaml --output /output --lang zh --video-sample-fps 4

说明：双相机图片文件夹标定；`/ABS/multi_cam_input` 下面必须有 `cam0` 和 `cam1` 两个子目录，每个目录图片数量必须一致，文件排序后按同一帧序号配对，视频输入不支持多相机模式。
docker run --rm -v /ABS/multi_cam_input:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --models pinhole-radtan,pinhole-radtan --lang zh

说明：三相机图片文件夹标定；`/ABS/three_cam_input` 下面必须有 `cam0`、`cam1`、`cam2`，每个相机同名或同序号图片数量一致，模型参数按相机顺序传入。
docker run --rm -v /ABS/three_cam_input:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --models pinhole-radtan,pinhole-radtan,pinhole-radtan --lang zh

说明：对输入图像做 CLAHE 预处理后再标定；当 AprilTag 角点提取受曝光、局部对比度或暗角影响时使用，报告会保留预处理后的 dataset，便于复查。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output_clahe:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --preprocess clahe --lang zh

说明：强制把所有图片 crop+resize 到 1280x720 后再标定；当输入分辨率或宽高比不一致时建议显式指定尺寸，否则 wrapper 会 warning 并按自动策略归一化。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output_resize:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --resize 1280x720 --lang zh

说明：给 Kalibr 指定初始焦距；当日志中出现初始化不稳定、优化发散、或默认 `Initializing focal length to 500` 明显不符合高分辨率相机时使用。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output_focal:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --focal-length-init 2400 --lang zh

说明：cam-cam 默认推荐的快优先模式；先用 Kalibr 多进程角点提取提升速度，如果日志检测到 rosbag 多进程读取异常，会自动归档 fast 结果并用单线程重跑，最终报告会写明 fallback 是否发生。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output_auto:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --fast-extraction auto --lang zh

说明：强制 cam-cam 多进程快跑；适合本机高速磁盘、数据量很大且你愿意自己承担风险的场景，wrapper 发现 rosbag 读取异常会直接报 error，不会自动单线程兜底。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output_fast:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --fast-extraction always --lang zh

说明：强制 cam-cam 单线程最稳模式；等价于给 Kalibr 加 `--no-multithreading`，适合共享盘、Parallels/虚拟机挂载目录、曾经出现 rosbag 读取异常或追求确定性复现的场景。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output_stable:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --fast-extraction never --lang zh

说明：verbose 模式会把 Kalibr 原始输出同步打印到终端，并保存更多中间文件和角点覆盖图；适合调试角点提取、覆盖不足、边缘缺失和优化失败。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output_verbose:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --verbose --lang zh

说明：只整理数据和做图像/角点诊断，不运行 Kalibr；适合先检查图片数量、分辨率归一化、角点覆盖、上下左右和四角是否缺样，再决定是否重新采集。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output_dryrun:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --skip-kalibr --verbose --lang zh

说明：英文报告和英文 warning/error；默认是中文，给外部英文用户或 CI 日志使用时加 `--lang en`。
docker run --rm -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output_en:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --lang en

======================================================================
cam-imu：corner-file 模式与 H5 模式
======================================================================

说明：cam-imu 的 ProductionCalibration 兼容 corner-file 模式；`corner-file` 是预提取角点 pkl，`image-timestamp-file` 是图像时间戳，`imu-data-file` 是对应 fixture 的 IMU CSV/TXT，输出目录会包含 `work_cam_imu`、Kalibr 结果和诊断报告。
docker run --rm -v /ABS/cam_imu_data:/data:ro -v /ABS/output_cam_imu:/output kalibr-camera-calibration:20.04 cam-imu --target /data/aprilgrid.yaml --cam-chain /data/camchain.yaml --imu-yaml /data/imu.yaml --corner-file /data/corners.pkl --image-timestamp-file /data/image_timestamps.txt --imu-data-file /data/imu.csv --fixture-id 1 --output /output --lang zh

说明：cam-imu 的 H5 输入模式；当图像数据已经被打成 H5，并且有图像时间戳和 Kalibr 兼容 IMU CSV 时使用。
docker run --rm -v /ABS/cam_imu_h5_data:/data:ro -v /ABS/output_cam_imu_h5:/output kalibr-camera-calibration:20.04 cam-imu --target /data/aprilgrid.yaml --cam-chain /data/camchain.yaml --imu-yaml /data/imu.yaml --h5-file /data/images.h5 --h5-timestamp-file /data/image_timestamps.txt --imu-csv /data/imu.csv --output /output --lang zh

说明：cam-imu 开启时间偏移估计；wrapper 默认关闭时间 offset 标定，适合先固定时间同步做稳健检查；如果要匹配 ProductionCalibration `private/zc-shuangzi-debug` 的 cam-imu 脚本，需要显式加 `--estimate-time-offset --timeoffset-padding 0.04 --no-export-poses`。
docker run --rm -v /ABS/cam_imu_data:/data:ro -v /ABS/output_cam_imu_time:/output kalibr-camera-calibration:20.04 cam-imu --target /data/aprilgrid.yaml --cam-chain /data/camchain.yaml --imu-yaml /data/imu.yaml --corner-file /data/corners.pkl --image-timestamp-file /data/image_timestamps.txt --imu-data-file /data/imu.csv --fixture-id 1 --estimate-time-offset --output /output --lang zh

说明：cam-imu 严格匹配 ProductionCalibration `private/zc-shuangzi-debug` 分支的离线脚本参数；该分支使用 `dataN.csv`、`--timeoffset-padding 0.04`、开启 temporal calibration、不导出 poses，适合做 Docker 与旧流程数值一致性验证。
docker run --rm -v /ABS/cam_imu_data:/data:ro -v /ABS/output_cam_imu_pcflags:/output kalibr-camera-calibration:20.04 cam-imu --target /data/aprilgrid.yaml --cam-chain /data/cam0-camchain-640x400.yaml --imu-yaml /data/imu.yaml --corner-file /data/cam0_640x400_corners.pkl --image-timestamp-file /data/0_save_timestamp.txt --imu-data-file /data/data1.csv --fixture-id 1 --output /output --lang zh --estimate-time-offset --timeoffset-padding 0.04 --no-export-poses

说明：cam-imu verbose 模式会把 Kalibr 原始日志同步打印到终端；适合检查 IMU 激励不足、gyro/accel residual 异常、时间偏移不稳定、优化失败等问题。
docker run --rm -v /ABS/cam_imu_data:/data:ro -v /ABS/output_cam_imu_verbose:/output kalibr-camera-calibration:20.04 cam-imu --target /data/aprilgrid.yaml --cam-chain /data/camchain.yaml --imu-yaml /data/imu.yaml --corner-file /data/corners.pkl --image-timestamp-file /data/image_timestamps.txt --imu-data-file /data/imu.csv --fixture-id 1 --output /output --verbose --lang zh

说明：cam-imu 快速预检模式；把最大迭代次数降到 10，只用于看残差趋势、时间戳是否对齐、外参是否明显跑飞，不能直接替代正式结果。
docker run --rm -v /ABS/cam_imu_data:/data:ro -v /ABS/output_cam_imu_fastcheck:/output kalibr-camera-calibration:20.04 cam-imu --target /data/aprilgrid.yaml --cam-chain /data/camchain.yaml --imu-yaml /data/imu.yaml --corner-file /data/corners.pkl --image-timestamp-file /data/image_timestamps.txt --imu-data-file /data/imu.csv --fixture-id 1 --output /output --max-iter 10 --no-export-poses --lang zh

说明：cam-imu 降低样条 knot rate 的提速实验命令；`pose/bias knots per second` 会改变优化变量数量和运动模型自由度，只适合和默认 `100/50` 对比后再决定是否采用。
docker run --rm -v /ABS/cam_imu_data:/data:ro -v /ABS/output_cam_imu_knots:/output kalibr-camera-calibration:20.04 cam-imu --target /data/aprilgrid.yaml --cam-chain /data/camchain.yaml --imu-yaml /data/imu.yaml --corner-file /data/corners.pkl --image-timestamp-file /data/image_timestamps.txt --imu-data-file /data/imu.csv --fixture-id 1 --output /output --pose-knots-per-second 60 --bias-knots-per-second 30 --no-export-poses --lang zh

======================================================================
报告查看、错误定位、结果复查
======================================================================

说明：查看 Markdown 报告前 220 行；报告里包含输入摘要、warning/error、角点覆盖评价、Kalibr 结果摘要和下一步采集建议。
sed -n '1,220p' /ABS/output/calibration_report.md

说明：查看 JSON 报告，适合脚本化读取 reprojection error、coverage、warning/error code、结果文件路径等结构化信息。
python3 -m json.tool /ABS/output/calibration_report.json | sed -n '1,260p'

说明：在报告和 Kalibr 日志里快速搜索严重错误；重点关注 `Did not converge`、`Optimization diverged`、`ImportError`、`NaN`、`matrix not positive definite` 等。
rg -n "ERROR|WARNING|Traceback|ImportError|NaN|Inf|Did not converge|Optimization diverged|matrix not positive definite" /ABS/output/calibration_report.md /ABS/output/kalibr_*.log

说明：查看 cam-cam 输出目录的核心产物；至少应该看到 `dataset`、`cam.bag`、`kalibr_cam_cam.log`、`calibration_report.md`、`calibration_report.json`，成功标定还会有 Kalibr 的 camchain/results/report 文件。
find /ABS/output -maxdepth 2 -type f | sort

说明：查看 cam-imu 输出目录的核心产物；`work_cam_imu` 内是为 Kalibr staging 的可写数据副本，顶层 report/json/log 是 wrapper 的诊断入口。
find /ABS/output -maxdepth 3 -type f | sort

======================================================================
Windows / Ubuntu 路径写法
======================================================================

说明：Ubuntu 原生 Docker 路径示例；宿主机绝对路径直接挂载到容器，命令形式和 macOS 一致。
docker run --rm -v /home/wayne/data/images:/input:ro -v /home/wayne/data/april_6x6.yaml:/target.yaml:ro -v /home/wayne/output:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --lang zh

说明：Windows PowerShell 路径示例；使用 `C:\...` 作为宿主机路径，冒号后的 `/input`、`/target.yaml`、`/output` 是容器内 Linux 路径。
docker run --rm -v C:\data\images:/input:ro -v C:\data\april_6x6.yaml:/target.yaml:ro -v C:\data\output:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --lang zh

说明：Apple Silicon 或其他非 amd64 机器上显式指定 linux/amd64；ROS Noetic/Kalibr 镜像按 amd64 构建时可避免 Docker 的平台提示变成误解，实际会通过 Docker Desktop/仿真运行。
docker run --rm --platform linux/amd64 -v /ABS/images:/input:ro -v /ABS/april_6x6.yaml:/target.yaml:ro -v /ABS/output:/output kalibr-camera-calibration:20.04 cam-cam --input /input --target /target.yaml --output /output --lang zh
