# 第 11 章：expression graph 与源码对应

前面章节已经把主要 residual 和 Jacobian 推出来了。读源码时还有一个障碍：Kalibr 并不在一个函数里手写完整 Jacobian。它把预测量写成一个 **expression graph**，每个节点只知道自己的值和局部 Jacobian，最后由后端按链式法则把这些局部 Jacobian 乘起来。

这一章不引入新的数学模型。它把第 4-10 章的公式对应到源码结构：

$$
\text{design variable}
\longrightarrow
\text{expression node}
\longrightarrow
\text{error term}
\longrightarrow
\text{JacobianContainer}
\longrightarrow
\mathbf H,\mathbf b_{\mathrm{rhs}}.
$$

读完这一章，应该能回答三件事：

1. 一个 residual 的 forward prediction 在源码里由哪些 expression node 组成。
2. 前面推导的局部 Jacobian 对应哪个 node 的 `evaluateJacobians(...)`。
3. 几何 residual/Jacobian 怎样被 information matrix、robust weight 和 design-variable scaling 变成线性系统里的 block。

## 11.1 expression graph 是什么

Expression graph 可以先理解成一棵计算树。叶子是 design variable 或常量，中间节点是代数操作，根节点是 residual 需要的预测量。

例如普通 gyro prediction：

$$
\hat{\boldsymbol\omega}
=
\mathbf R_{ib}\boldsymbol\omega_b(t)
+
\mathbf b_g(t)
$$

可以看成：

```text
pose spline controls  -> angularVelocityBodyFrame(t) -> omega_b
q_i_b_Dv              -> RotationExpression        -> R_ib
gyro bias controls    -> EuclideanBSplineExpression -> b_g

R_ib * omega_b  -> EuclideanExpressionMultiply
(...) + b_g     -> EuclideanExpressionAdd
```

每个节点做两件事：

1. `toEuclidean()` / `toRotationMatrix()` / `toTransformationMatrix()` 计算当前值。
2. `evaluateJacobians(outJacobians, applyChainRule)` 把上游链式矩阵继续传给子节点。

`applyChainRule` 是第 3 章链式法则里的“上游 Jacobian”。如果当前节点输出 $\mathbf y$，父节点已经给出：

$$
\mathbf A
=
\frac{\partial\mathbf e}{\partial\mathbf y},
$$

而当前节点对某个子节点 $\mathbf x$ 的局部 Jacobian 是：

$$
\mathbf J_{\mathbf y,\mathbf x}
=
\frac{\partial\mathbf y}{\partial\mathbf x},
$$

那么传给子节点的就是：

$$
\boxed{
\mathbf A\mathbf J_{\mathbf y,\mathbf x}.
}
$$

源码里的 `applyChainRule * localJacobian` 就是这个乘法。

## 11.2 Design variable 和 tangent block

Kalibr 优化的不是矩阵元素本身，而是每个 design variable 的 minimal tangent block。

| 数学对象 | 典型 design variable | minimal dimension |
|---|---|---:|
| scalar time shift $\Delta t$ | `Scalar` | $1$ |
| Euclidean point / bias control point | `EuclideanPointDv` / spline DV | $3$ |
| rotation | `RotationQuaternionDv` | $3$ |
| transformation | `TransformationDv` | $6$ |
| matrix with mask | `MatrixBasicDv` | active entries count |
| pose spline control point | `BSplinePoseDesignVariable` coefficient | $6$ |

因此，expression node 输出的 Jacobian 列数不是“矩阵有多少元素”，而是对应 design variable 的 minimal dimension。例如：

$$
\mathbf R_{ib}\in SO(3)
$$

在源码里对应 $3$ 维 rotation tangent，所以第 8 章的外参旋转 Jacobian 是 $3\times3$，不是 $3\times9$。而 `MatrixBasicDv` 是普通矩阵参数，它的 full Jacobian 先是 $3\times9$，再由 mask 选取 active columns。

## 11.3 Camera residual 的 expression graph

Camera residual 的源码主线在 `IccSensors.py`：

```python
frameTime = cameraTimeToImuTimeDv.toExpression() \
          + obs.time().toSec() \
          + timeshiftCamToImuPrior

T_w_b = poseSplineDv.transformationAtTime(frameTime, ...)
T_b_w = T_w_b.inverse()
T_c_w = T_cN_b * T_b_w
p = T_c_w * aopt.HomogeneousExpression(targetPoint)
rerr = error_t(imagePoint, invR, p, frame)
```

对应的数学链路是：

$$
t
=
t^{\mathrm{cam}}+\Delta t^{\mathrm{prior}}+\Delta t,
$$

$$
\mathbf T_{c_nw}(t)
=
\mathbf T_{c_nb}\mathbf T_{wb}(t)^{-1},
$$

$$
\tilde{\mathbf p}_{c_n}
=
\mathbf T_{c_nw}\tilde{\mathbf p}_w,
$$

$$
\mathbf e^\pi
=
\mathbf y-\boldsymbol\pi(\tilde{\mathbf p}_{c_n}).
$$

Reprojection error 的 residual 方向在源码里很明确：

```cpp
cam.homogeneousToKeypoint(p, hat_y, J);
setError(_y - hat_y);
_point.evaluateJacobians(_jacobians, -J);
```

所以第 4 章 camera residual 的负号来自：

$$
\mathbf e^\pi=\mathbf y-\hat{\mathbf y}.
$$

后面的 transform/action/product/inverse Jacobian 都是在这个 $-J$ 上继续乘局部链式矩阵。

## 11.4 IMU residual 的 expression graph

Gyro 和 accelerometer residual 使用 `EuclideanError`。它的源码方向是：

```cpp
Eigen::VectorXd errorVector = _t.toEuclidean() - _prior;
setError(errorVector);
_t.evaluateJacobians(_jacobians);
```

其中 `_t` 是预测 expression，`_prior` 是 measurement。因此：

$$
\mathbf e^\omega=\hat{\boldsymbol\omega}-\mathbf z^\omega,
\qquad
\mathbf e^a=\hat{\mathbf a}-\mathbf z^a.
$$

普通 gyro 的 Python expression 是：

```python
w_b = poseSplineDv.angularVelocityBodyFrame(tk)
b_i = self.gyroBiasDv.toEuclideanExpression(tk, 0)
C_i_b = self.q_i_b_Dv.toExpression()
w = C_i_b * w_b
gerr = ket.EuclideanError(im.omega, im.omegaInvR * weight, w + b_i)
```

它对应：

$$
\hat{\boldsymbol\omega}
=
\mathbf R_{ib}\boldsymbol\omega_b(t_k)+\mathbf b_g(t_k).
$$

普通 accelerometer 的 Python expression 是：

```python
C_b_w = poseSplineDv.orientation(tk).inverse()
a_w = poseSplineDv.linearAcceleration(tk)
b_i = self.accelBiasDv.toEuclideanExpression(tk, 0)
w_b = poseSplineDv.angularVelocityBodyFrame(tk)
w_dot_b = poseSplineDv.angularAccelerationBodyFrame(tk)
C_i_b = self.q_i_b_Dv.toExpression()
r_b = self.r_b_Dv.toExpression()
a = C_i_b * (C_b_w * (a_w - g_w)
             + w_dot_b.cross(r_b)
             + w_b.cross(w_b.cross(r_b)))
aerr = ket.EuclideanError(im.alpha, im.alphaInvR * weight, a + b_i)
```

它对应第 7 章的：

$$
\hat{\mathbf a}
=
\mathbf R_{ib}
\left[
\mathbf R_{bw}(\mathbf a_w-\mathbf g_w)
+
\boldsymbol\alpha_b\times\mathbf r_b
+
\boldsymbol\omega_b\times
(\boldsymbol\omega_b\times\mathbf r_b)
\right]
+
\mathbf b_a.
$$

## 11.5 局部 node Jacobian 速查

这一节把常见 expression node 和前面推导过的局部 Jacobian 放在一起。表里的 $\mathbf A$ 表示传入当前节点的 `applyChainRule`。

| node | 输出 | 子节点局部 Jacobian |
|---|---|---|
| add | $\mathbf x+\mathbf y$ | 对两边都是 $\mathbf I$ |
| subtract | $\mathbf x-\mathbf y$ | 对左边 $\mathbf I$，对右边 $-\mathbf I$ |
| negated | $-\mathbf x$ | $-\mathbf I$ |
| rotation-vector multiply | $\mathbf R\mathbf x$ | 对 $\mathbf x$ 是 $\mathbf R$，对 $\delta\boldsymbol\phi_K$ 是 $[\mathbf R\mathbf x]_\times$ |
| matrix-vector multiply | $\mathbf M\mathbf x$ | 对 $\mathbf x$ 是 $\mathbf M$，对 full $\operatorname{vec}(\mathbf M)$ 是 $[x_1\mathbf I\ x_2\mathbf I\ x_3\mathbf I]$ |
| cross | $\mathbf a\times\mathbf b$ | 对 $\mathbf a$ 是 $-[\mathbf b]_\times$，对 $\mathbf b$ 是 $[\mathbf a]_\times$ |
| transform-point | $\mathbf T\tilde{\mathbf p}$ | 对 $\mathbf T$ 是 `boxMinus(Tp)`，对点是 $\mathbf T$ |
| transform product | $\mathbf A\mathbf B$ | 对左边 identity，对右边 `boxTimes(A)` |
| transform inverse | $\mathbf T^{-1}$ | 对原 transform 是 `-boxTimes(T^{-1})` |
| rotation product | $\mathbf R_1\mathbf R_2$ | 对左边 identity，对右边 $\mathbf R_1$ |
| rotation inverse | $\mathbf R^\top$ | 对原 rotation 是 $-\mathbf R^\top$ |
| time-offset pose spline | $\mathbf T(\Delta t)$ | 对 time scalar 是 $\mathbf J_{\mathbf T,\mathbf s}\dot{\mathbf s}$ |

源码对应关系：

| node | 文件 |
|---|---|
| rotation action、matrix-vector、cross、add/sub | `aslam_backend_expressions/src/EuclideanExpressionNode.cpp` |
| transform-point | `aslam_backend_expressions/src/HomogeneousExpressionNode.cpp` |
| transform product / inverse | `aslam_backend_expressions/src/TransformationExpressionNode.cpp` |
| rotation product / inverse | `aslam_backend_expressions/src/RotationExpressionNode.cpp` |
| time-offset pose spline | `aslam_splines/src/BSplineExpressions.cpp` |

## 11.6 `applyChainRule` 怎样沿树传播

以 matrix-vector node 为例：

$$
\mathbf y=\mathbf M\mathbf x.
$$

如果上游给出：

$$
\mathbf A=\frac{\partial\mathbf e}{\partial\mathbf y},
$$

当前节点会给 matrix 子节点：

$$
\mathbf A
\begin{bmatrix}
x_1\mathbf I_3&
x_2\mathbf I_3&
x_3\mathbf I_3
\end{bmatrix},
$$

给 vector 子节点：

$$
\mathbf A\mathbf M.
$$

源码正是：

```cpp
_lhs->evaluateJacobians(outJacobians, applyChainRule * J_full);
_rhs->evaluateJacobians(outJacobians, applyChainRule * _A_lhs);
```

这说明 expression graph 的工作方式不是“每个节点知道最终 residual 的完整公式”，而是每个节点只处理自己这一步的链式传播。复杂 residual 之所以能自动求导，是因为所有局部节点都遵守同一个接口。

## 11.7 从 JacobianContainer 到 Hessian

普通 error term 的线性系统装配路径是：

```cpp
JacobianContainer J(C);
evaluateJacobians(J);
sqrtWeight = sqrt(mEstimator.getWeight(rawSquaredError));
J.evaluateHessian(_error, sqrtWeight * _sqrtInvR, outHessian, outRhs);
```

设几何 residual 和几何 Jacobian 为：

$$
\mathbf e,
\qquad
\mathbf J_i=\frac{\partial\mathbf e}{\partial\delta\boldsymbol\theta_i}.
$$

Information matrix 的平方根在源码中满足：

$$
\boldsymbol\Omega
=
\mathbf L\mathbf L^\top,
\qquad
\mathbf L=\texttt{sqrtInvR}.
$$

Robust kernel 给出标量权重 $w$。Design variable 还有一个 scaling $s_i$，cam-imu 主路径里通常是 $1$。真正用于 Hessian 的 residual/Jacobian 是：

$$
\boxed{
\bar{\mathbf e}
=
\sqrt{w}\,\mathbf L^\top\mathbf e,
\qquad
\bar{\mathbf J}_i
=
s_i\sqrt{w}\,\mathbf L^\top\mathbf J_i.
}
$$

于是每个 error term 对 block normal equation 的贡献是：

$$
\boxed{
\mathbf H_{ij}
\mathrel{+}=
\bar{\mathbf J}_i^\top
\bar{\mathbf J}_j,
\qquad
\mathbf b_i
\mathrel{-}=
\bar{\mathbf J}_i^\top
\bar{\mathbf e}.
}
$$

这就是第 3 章的优化公共层在源码里的落点。前面章节推导的都是 $\mathbf J_i$，也就是未加 whitening、robust scaling 和 design-variable scaling 的几何 Jacobian。

## 11.8 Motion prior 的例外

第 9 章的 `BSplineMotionError` 是例外。它不显式调用普通 `evaluateJacobians(...)` 路径，而是把：

$$
E=\mathbf c^\top\mathbf Q\mathbf c
$$

直接装进线性系统：

$$
\mathbf H\mathrel{+}=\mathbf Q,
\qquad
\mathbf b_{\mathrm{rhs}}\mathrel{-}=\mathbf Q\mathbf c.
$$

源码中 `evaluateJacobiansImplementation(...)` 甚至会抛异常。原因不是它没有 Jacobian，而是它采用二次型直接装配。若需要把它放进普通 residual/Jacobian 语言，可以选择 $\mathbf Q=\mathbf S^\top\mathbf S$，令 $\mathbf e=\mathbf S\mathbf c$，$\mathbf J=\mathbf S$。

## 11.9 读源码的顺序

面对一个新的 Kalibr residual，可以按下面顺序读：

1. 找 error term 构造处，确认 measurement、prediction expression 和 information matrix。
2. 找 `evaluateErrorImplementation()`，确认 residual 方向是 prediction minus measurement，还是 measurement minus prediction。
3. 找 prediction expression 的根节点，沿着 expression tree 展开 forward model。
4. 对每个中间节点查本章 11.5 的局部 Jacobian。
5. 把局部 Jacobian 从根往叶子乘，得到几何 Jacobian。
6. 最后再加上 whitening、robust weight 和 design-variable scaling。

这个顺序能避免两个常见错误：把 residual 方向负号和 perturbation convention 负号混在一起；把源码里的 whitened Jacobian 当成几何 Jacobian。

## 11.10 本章小结

Kalibr 的核心实现方式是 expression graph。每个 node 只实现自己的 forward value 和局部 Jacobian，`applyChainRule` 把上游 residual Jacobian 继续传给子节点。Camera residual 的特殊负号来自 reprojection error 的 $\mathbf y-\hat{\mathbf y}$；gyro 和 accelerometer 的 `EuclideanError` 是 prediction minus measurement。

前面章节里的 `boxMinus`、`boxTimes`、rotation action、cross product、matrix-vector Jacobian，都能在 expression node 源码中找到对应局部实现。普通 error term 最终经过 information matrix、robust weight 和 design-variable scaling 进入 Hessian；motion prior 则直接用 $\mathbf Q$ 装配 Hessian 和 RHS。

附录 B 会把这些公式变成最小数值验证：用 finite difference 检查 SO(3)、SE(3)、projection、IMU residual 和 motion-prior block 的 analytic Jacobian。
